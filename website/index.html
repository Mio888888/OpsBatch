Why OpsBatch

运维工具够了，但碎片化太严重

SSH 客户端、文件传输、远程桌面、批量脚本、知识库散落在多个工具中。OpsBatch 把它们装进一个桌面应用，数据优先留在本机。

痛点
SSH / SFTP / RDP 各用各的软件，来回切窗口
OpsBatch
终端、文件、远程桌面放进同一套主机上下文
痛点
批量操作靠手动 for 循环，没有历史回放
OpsBatch
多主机批量执行 + 失败重试 + asciinema 回放
痛点
命令、脚本散落在 wiki / 笔记本，找不到也复用不了
OpsBatch
命令库 + 脚本库 + 快捷动作 + Git 仓库同步
痛点
操作审计靠云平台，数据不在自己手里
OpsBatch
本地 SQLite + 系统关键凭据 + 无云依赖即可运行
痛点
重复运维流程靠记忆，新人上手周期长
OpsBatch
可视化工作流编排 + 模板复用 + AI 辅助说明

Core Features

一站式运维能力

从资产管理到远程桌面，从批量执行到工作流编排，覆盖日常运维高频场景。

资产管理 核心基础

主机资产与分组

  • 主机名称、IP、端口、认证方式、操作系统、分组标签
  • 嵌套分组、资产搜索、批量选择
  • 解析 ~/.ssh/config 与 ProxyJump 跳板链路
  • CSV 导入导出 + 阿里云 / AWS / 腾讯云实例拉取
终端 xterm.js

SSH 终端工作区

  • 本地终端 + 远程 SSH 多标签页
  • Rust SSH 实现（russh），稳定低延迟
  • 主机监控：CPU、内存、磁盘、网络、进程快照
  • 端口转发面板 + 远程编辑器（CodeMirror）
文件传输 SFTP

SFTP 文件面板

  • 本地 / 远程双栏浏览
  • 上传、下载、预览、重命名、删除、书签
  • 传输队列 + 解压支持
  • 远程编辑器：从 SFTP 打开文件并保存回远端
远程桌面 RDP / VNC

RDP 与 VNC 远程桌面

  • Windows RDP：独立窗口连接，H.264 直显 / bitmap 双路径，支持分辨率 / 剪贴板 / 音频 / 磁盘映射
  • VNC：noVNC + WebSocket bridge，支持共享连接和只读模式
  • 连接、帧率、传输诊断日志，便于排查问题
  • 分辨率边界保护：VNC ≤ 1920×1080，RDP 640×480 ~ 3840×2160
AI 辅助 可选功能

AI 运维助手

  • 终端上下文 AI 问答与建议
  • 危险命令风险说明
  • 自定义 API 端点与模型
  • 仅用户主动发起时提交上下文
知识沉淀 命令 / 脚本

命令库 & 脚本库

  • 搜索、分类筛选、自定义命令和脚本
  • 风险等级、平台字段、远程 URL 命令
  • 快捷动作：参数占位符、JSON 导入导出
  • Git 仓库同步：定时拉取命令 / 脚本 / 快捷动作
批量执行 并发控制

批量命令与文件分发

  • 多主机并发执行：并发数、超时、实时结果、失败重试、历史记录
  • 危险命令防护：内置 + 自定义正则规则，高风险命令执行前确认，支持 AI 风险说明
  • 广播终端：独立窗口对多台主机广播输入（最多 16 台）
  • 批量文件分发：上传到多台主机，支持 {host} 变量
  • asciinema 格式执行回放

Workflow

可视化工作流编排

基于 React Flow 的拖拽式工作流编辑器。将日常运维流程沉淀为可复用、可审计的自动化工作流。支持条件分支、人工确认、回滚等节点类型。

01
选择主机 从资产分组中勾选目标主机
02
编排节点 拖拽命令、脚本、传输、条件分支等节点
03
配置变量 结果变量传递、超时重试、失败策略
04
执行 & 追踪 实时查看每步执行结果，支持回滚和重试
05
模板复用 保存为模板，一键应用到其他主机分组

Architecture

技术架构

前端 React 19 + TypeScript / Vite，后端 Rust / Tauri v2。核心数据持久化在本机 SQLite 数据库，通过 Tauri IPC 桥接前后端。

React 19 + TypeScript 终端 · SFTP · RDP · VNC · 工作流 · AI
Tauri IPC 命令桥接
Rust 后端 SSH · RDP · VNC · SQLite · PTY
前端

React 19 + Vite

Radix UI 组件基座，Zustand 状态管理，xterm.js 终端，CodeMirror 编辑器，React Flow 工作流画布。

后端

Rust / Tauri v2

russh SSH 实现，ironrdp 远程桌面，rusqlite + r2d2 连接池，git2 仓库同步，本地 PTY。

数据

本地优先

SQLite 持久化（r2d2 连接池），系统 keychain 凭据保护，无需云服务即可完整运行。

部署

跨平台安装包

macOS .dmg / .app.tar.gz，Windows .exe / .msi，Linux .AppImage / .deb / .rpm。GitHub Releases 自动发布。

Security

安全与隐私

OpsBatch 是本地优先的桌面应用，不依赖中心服务。主机资产、凭据、执行历史和工作流主要保存在本地 SQLite 和系统 keychain 中。

🔐

本地优先架构

核心数据保存在本机 SQLite 数据库，不依赖云服务即可运行。部分凭据尽力写入系统 keychain。

🛡️

危险命令防护

内置和自定义正则规则，高风险命令执行前触发确认。预编译正则消除终端按键时的重复编译开销。支持 AI 辅助风险说明。

🔑

SSH 原生连接

基于 Rust SSH 实现（russh），解析 ~/.ssh/config 和 ProxyJump 跳板链路。连接信号量限制单次并发为 4。

📡

AI 可选且受控

AI 功能需用户自行配置端点和 API Key，终端上下文仅在用户主动发起时提交。AI 建议仅供参考，用户确认后执行。
